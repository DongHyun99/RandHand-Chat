## 커밋 메시지 규칙 

1. 문장의 끝에 `.` 를 붙이지 말기

2. 이슈 번호를 커밋 메시지 끝에 붙이기

3. 형식

   > [타입]: [내용] [이슈 번호]

4. 예시

   > docs: OO메소드 관련 설명 주석 [#3]
   >
   > feature: 예약 시스템의 add() [#6]

5. 타입 종류

   > \- chore : 간단한 수정
   >
   > \- feature : 새로운 기능
   >
   > \- fix : 버그 대처
   >
   > \- refactor : 코드 수정 / 리팩터링
   >
   > \- test : 테스트 추가
   >
   > \- docs : 문서 작성
